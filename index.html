Both are in Supabase → Project Settings → API. The anon key is safe to put here; what it can do is controlled by your table policies.

First time? Create the tables (run this SQL once)

In Supabase, open SQL Editor → New query, paste this, and click Run. It creates all the tables and opens them to your anon key.

Schema SQL 
-- WealthWise schema for Supabase
-- Note: the in-app "Secure Vault" is intentionally NOT here — it stays on-device and is never synced.
-- Run once: Supabase dashboard -> SQL Editor -> New query -> paste -> Run. Safe to re-run.

create table if not exists transactions (
  id text primary key, type text, amount numeric, category text, description text, date date,
  committed boolean default false, "committedBy" text, "committedAt" timestamptz,
  "createdBy" text, "createdByUser" text, created_at timestamptz default now()
);
create table if not exists investments (
  id text primary key, name text, type text, invested numeric, current numeric, "startDate" date,
  "createdBy" text, "createdByUser" text, created_at timestamptz default now()
);
create table if not exists loans (
  id text primary key, name text, principal numeric, rate numeric, tenure int, "startDate" date, type text,
  "createdBy" text, "createdByUser" text, created_at timestamptz default now()
);
create table if not exists budgets (
  id text primary key, category text, "limit" numeric,
  "createdBy" text, "createdByUser" text, created_at timestamptz default now()
);
create table if not exists goals (
  id text primary key, name text, category text, target numeric, current numeric, date date,
  "createdBy" text, "createdByUser" text, created_at timestamptz default now()
);
create table if not exists documents (
  id text primary key, title text, category text, "refNo" text, "issuedOn" date, "expiresOn" date,
  link text, notes text, "fileData" text, "fileName" text, "fileType" text,
  "createdBy" text, "createdByUser" text, created_at timestamptz default now()
);
create table if not exists assets (
  id text primary key, name text, type text, value numeric, notes text,
  "createdBy" text, "createdByUser" text, created_at timestamptz default now()
);
create table if not exists liabilities (
  id text primary key, name text, type text, amount numeric, notes text,
  "createdBy" text, "createdByUser" text, created_at timestamptz default now()
);
create table if not exists contributions (
  id text primary key, type text, category text, amount numeric, description text, date date,
  "createdBy" text, "createdByUser" text, created_at timestamptz default now()
);
create table if not exists family (
  id text primary key, relation text, name text, dob date, phone text, occupation text, notes text,
  "createdBy" text, "createdByUser" text, created_at timestamptz default now()
);
create table if not exists lending (
  id text primary key, type text, person text, amount numeric, date date, "dueDate" date, note text, status text default 'pending',
  "createdBy" text, "createdByUser" text, created_at timestamptz default now()
);
create table if not exists messages (
  id text primary key, "fromUser" text, "fromName" text, "fromRole" text, "toUser" text,
  text text, ts timestamptz, read boolean default false,
  "fileData" text, "fileName" text, "fileType" text,
  "createdBy" text, "createdByUser" text, created_at timestamptz default now()
);

-- Open the tables to your public anon key (shared-book model).
-- WARNING: anyone with your project URL + anon key can then read/write this data.
do $$
declare t text;
begin
  foreach t in array array['transactions','investments','loans','budgets','goals','documents','assets','liabilities','contributions','family','lending','messages'] loop
    execute format('alter table %I enable row level security;', t);
    execute format('drop policy if exists ww_all on %I;', t);
    execute format('create policy ww_all on %I for all to anon, authenticated using (true) with check (true);', t);
  end loop;
end $$;

After connecting the first time, click Upload local data once to seed the database with what you already have. After that it stays in sync on its own and refreshes from the database every 20s.
